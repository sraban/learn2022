Q. When States are inter-dependent - don't modify in same function ... updating each state takes sometime to render the component. ------carefully handle the states

   Ex. 
   	function Component({}) {
      const calculation = () => {
      	setAddress()
      	setContact()
      }
    }

   Sol. Works one function after another synchronously -- states are updated one after other with tpl render each
    function Component({}) {
      const calculation = () => {
      	setAddress()
      	contact();
      }
      
      const contact = () => {
      	setContact();
      }
    }

What is React?
	React is a JavaScript library for building fast and interactive user interfaces.
	So, every React application is essentially a tree of components.
	So, React keeps a light-weight representation of the DOM in memory
	React and Angular are similar in terms of their component-based architecture
	But Angular is a framework or a complete solution, while React is a library
