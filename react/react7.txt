Q. When States are inter-dependent - don't modify in same function ... updating each state takes sometime to render the component. ------carefully handle the states

   Ex. 
   	function Component({}) {
      const calculation = () => {
      	setAddress()
      	setContact()
      }
    }

   Sol. Works one function after another synchronously -- states are updated one after other with tpl render each
    function Component({}) {
      const calculation = () => {
      	setAddress()
      	contact();
      }
      
      const contact = () => {
      	setContact();
      }
    }

Q. React.js Commonly Faced Problems

1 — Not starting a component name with a capital letter <Component> not like <div>
2 — Using single quotes instead of back-ticks
3— Using React.PropTypes
4 — Not using the right versions of what a tutorial is using
	React.Component, not React.component
	componentDidMount, not ComponentDidMount
	ReactDOM, not ReactDom
	
5— Confusing functions with classes
6 — Passing numbers as strings
7 — Forgetting that another app instance is still using the same port
8 — Forgetting to create environment variables
9— Confusing curly braces {} with parentheses ()
10 — Not wrapping objects with parentheses
11 — Not using the right capitalization of API elements and props
12— Confusing the state object with instance properties
13 — Confusing <tag/> with </tag>
14 — Assuming import/export will just work
15 — Not binding handler methods

What is React?
	React is a JavaScript library for building fast and interactive user interfaces.
	So, every React application is essentially a tree of components.
	So, React keeps a light-weight representation of the DOM in memory
	React and Angular are similar in terms of their component-based architecture
	But Angular is a framework or a complete solution, while React is a library
